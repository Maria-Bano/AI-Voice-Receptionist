# Contributing

Thank you for your interest in this project.

## Repository Type

This is a **public portfolio repository** — not an open-source software project seeking feature contributions. It exists to showcase architecture, engineering decisions, and production thinking for recruiters, clients, employers, and Upwork prospects.

## What We Accept

Contributions are **limited to documentation improvements**, such as:

- Typo and grammar fixes in README, case study, or architecture docs
- Clarity improvements to Mermaid diagrams or explanations
- Broken link fixes
- Accessibility or formatting improvements in markdown
- Translation of documentation (with maintainer approval)

## What We Do Not Accept

Please do **not** open pull requests that include:

- Production n8n workflow JSON exports
- API keys, credentials, webhook URLs, or `.env` files
- Proprietary prompts, system instructions, or tuning details
- Proprietary business logic or operational rules
- Customer, guest, or reservation data
- Unsanitized screenshots from production systems
- New feature code or production implementation details

If you are interested in collaboration on a production build, contact the maintainer directly via the links in [README.md](README.md#contact).

## How to Contribute

1. Fork the repository
2. Create a branch for your documentation change
3. Make your edits — documentation files only
4. Review your diff for accidental secrets or proprietary content
5. Open a pull request using the [PR template](.github/PULL_REQUEST_TEMPLATE.md)
6. Describe what you changed and why

## Review Process

- Documentation PRs are reviewed for accuracy, clarity, and security (no secrets)
- PRs that add proprietary or credential content will be closed without merge
- Maintainers may edit or request changes before merging

## Code of Conduct

Be respectful and professional. This is a portfolio project shared in a professional context.

## Security

If you find a security issue — including accidentally exposed credentials — do **not** open a public issue. See [SECURITY.md](SECURITY.md) for the responsible disclosure process.

## Questions

Open a [documentation issue](.github/ISSUE_TEMPLATE.md) for questions about the architecture or case study. For everything else, use the contact links in the README.
